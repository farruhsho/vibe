### АКТУАЛЬНОСТЬ ТЕМЫ

В современную эпоху цифровых музыкальных платформ пользователи сталкиваются с проблемой информационной перегрузки: каталоги стриминговых сервисов содержат десятки миллионов треков, что делает ручной поиск музыки по вкусу крайне затруднительным. Существующие рекомендательные системы (Spotify, Apple Music, YouTube Music) используют преимущественно коллаборативную фильтрацию и готовые плейлисты, которые ориентированы на усредненные музыкальные предпочтения широкой аудитории.

Основная проблема современных подходов заключается в отсутствии глубокого анализа индивидуальных паттернов прослушивания пользователя. Системы не учитывают уникальные предпочтения по аудио-характеристикам (энергия, валентность, танцевальность, темп), которые формируют музыкальный вкус конкретного человека. Это приводит к низкой точности рекомендаций и необходимости постоянной ручной корректировки плейлистов.

Данная работа направлена на решение указанной проблемы путем разработки алгоритма рекомендации музыки, основанного на выявлении и применении индивидуальных паттернов прослушивания через анализ аудио-характеристик треков.

## ANNOTATION (English)

**Title:** Pattern-Based Music Recommendation Algorithm Using Audio Feature Analysis

**Keywords:** recommendation systems, music analysis, pattern algorithm, audio features, personalization, machine learning, Flutter, Spotify API, Firebase

**Relevance:** Modern music streaming platforms contain millions of tracks, making manual music discovery difficult. Existing recommendation systems use generic playlists that don't analyze individual listening patterns. This work proposes a pattern-based algorithm that identifies user preferences through audio features (energy, tempo, valence, danceability) and applies personalized weights for recommendations.

**Aim:** To develop a pattern-based music recommendation algorithm that selects tracks by analyzing user listening patterns and audio features.

**Methods:** Literature analysis, mathematical modeling (Gaussian distribution, weighted scoring), algorithm design, experimental testing with Spotify Web API, mobile app prototyping with Flutter, statistical analysis of user ratings.

**Results:**
1. Designed algorithm architecture for pattern recognition and recommendation scoring
2. Implemented weighted scoring system: energy (30%), valence (30%), danceability (25%), tempo (15%)
3. Developed mobile application "Vibe" for iOS/Android with Spotify API integration
4. Created analytics dashboard for pattern visualization
5. Implemented 5-star rating system for hypothesis validation (target: ≥4.0/5.0 average)

**Scientific novelty:** Novel pattern-based approach using Gaussian distribution for feature matching, dynamic pattern learning that adapts to user taste, pattern strength metric for preference consistency measurement.

**Practical significance:** Working mobile app prototype, improved recommendation accuracy through individual pattern analysis, transparent recommendations with match percentages, open architecture for further research.
